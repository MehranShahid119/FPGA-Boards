first time setup
New Project → GitHub
# 1. Connect to GitHub
git remote add origin https://github.com/user/repo.git

# 2. Rename branch (if needed)
git branch -m master main

# 3. Merge remote history
git pull origin main --allow-unrelated-histories --no-rebase

# 4. Push & set upstream
git push -u origin main
daily usage
Regular Push Workflow
# Stage all changes
git add .

# Commit with message
git commit -m "your message here"

# Push (upstream already set)
git push
wsl dns fix
github.com Not Resolving
# Block auto-override
sudo nano /etc/wsl.conf
# Add: [network] generateResolvConf = false

# Set Google DNS
sudo rm /etc/resolv.conf
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
sudo chattr +i /etc/resolv.conf

# Restart WSL (in PowerShell)
wsl --shutdown

01
Empty repo on GitHub — When pushing an existing local project, create the GitHub repo without README/.gitignore to avoid merge conflicts.
02
Use .gitignore — Always add a .gitignore to keep sensitive files, large binaries, and temp files out of the repo.
03
Branch name consistency — Use either main or master everywhere — never mix both.
04
WSL network — If github.com won't resolve, run ping github.com first to confirm DNS issue before fixing.
05
Use SSH keys — Switch from HTTPS to SSH to avoid entering your Personal Access Token every time you push.
