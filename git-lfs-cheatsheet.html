📦 Basic Git Workflow

Daily use commands — har roz kaam aane wale commands

Stage & Commit 
# Sab files stage karein
git add .

# Specific file/folder stage karein
git add path/to/file.txt
git add src/

# Commit with message
git commit -m "feat: add new feature"
git commit -am "fix: quick fix"

# Status check ← sabse zaroori!
git status
git status --short
Push & Pull 
# Pehli baar push (upstream set karein)
git push -u origin main
git push -u origin master

# Baad mein normal push
git push

# Remote changes pull karein
git pull origin main
git pull --rebase origin main

# Remote URL check/change
git remote -v
git remote set-url origin https://github.com/user/repo.git
02

🗄️ Git LFS Commands

Large files (>50MB) ke liye — CSV, MAT, BIN, H5, PT, PTH, PDF, PNG, ZIP

Setup & Tracking 
# LFS initialize (sirf ek baar per machine)
git lfs install

# Extensions ko LFS track karein
git lfs track "*.csv" "*.mat" "*.bin"
git lfs track "*.h5" "*.pt" "*.pth"
git lfs track "*.pdf" "*.png" "*.zip"

# Specific file track karein
git lfs track "datasets/large_file.bin"

# Track patterns dekhein
git lfs track

# .gitattributes commit karein ⚠️ ZAROORI
git add .gitattributes
git commit -m "chore: configure LFS tracking"
⚠️
.gitattributes hamesha commit karein — bina iske LFS rules doosre machines par apply nahi honge!
Verification & Push 
# Kaunsi files LFS ke under hain?
git lfs ls-files

# Pending LFS uploads
git lfs status

# LFS operation logs (debugging)
git lfs logs last

# File type verify karein
file path/to/large_file.bin
# ✅ LFS pointer: "ASCII text" + "version..."
# ❌ Normal Git: "data" ya actual file type

# Push (LFS auto handle karega)
git push

# LFS files alag se push
git lfs push origin main

# Sab LFS objects verify
git lfs fetch --all
git lfs checkout
03

🔧 Fix Commands

Jab cheezein galat ho jayen — recovery aur conversion commands

File Already in Git → Convert to LFS 
# Step 1: Sab files unstage karein
git reset

# Step 2: Git cache se hatao (disk se DELETE nahi)
git rm --cached path/to/large_file.bin

# Step 3: Dobara add (LFS intercept karega)
git add path/to/large_file.bin

# Step 4: Verify karein
git lfs ls-files | grep large_file
Wrong Extension / Stuck Process 
# Wrong extension fix
git lfs track "*.csv"   # sahi extension
git add .gitattributes
git commit -m "fix: correct LFS pattern"
git rm --cached *.csv
git add *.csv

# Git index lock delete karein
rm -f .git/index.lock

# Corrupt index repair
git update-index --really-refresh

# Force unlock (frozen terminal)
# Ctrl + C dabayein, ya naya terminal
🛑
git rm --cached sirf Git index se hata-ta hai — aapki actual file safe rehti hai disk par.

🌐 Remote & Branch Management

Remote URLs, branches, aur switching repos

Remote Operations 
# Naya remote add karein
git remote add origin https://github.com/user/repo.git

# Remote URL change karein
git remote set-url origin https://github.com/user/new-repo.git

# Remote remove karein
git remote remove origin

# Sab remotes dekhein
git remote -v
Branch Operations 
# Current branch dekhein
git branch

# Nayi branch banayein + switch
git checkout -b feature/new-model

# Branch rename (master → main)
git branch -M main

# Branch delete (local)
git branch -d old-branch
git branch -D old-branch  # force

# Sab branches dekhein (local + remote)
git branch -a

🔍 Troubleshooting & Debugging

Common errors aur diagnostic commands

Error Solution
file is X MB; exceeds 100 MB File LFS mein nahi → git lfs track "*.ext" + git add .gitattributes + git add <file>
fatal: Authentication failed GitHub PAT use karein — normal password nahi chalta
remote: Repository not found Repo URL check karein + access permissions verify karein
src refspec master does not match git branch -a se branch naam check karein, phir sahi se push
error: failed to push some refs Pehle git pull karein ya --force carefully use karein
Diagnostic Commands 
# Large files dhoondhein (>100MB)
find . -type f -size +100M -exec ls -lh {} \;

# Project total size
du -sh .

# Git objects size check
git count-objects -vH

# Last 5 commits dekhein
git log --oneline -5

# Staged changes preview
git diff --cached --stat

⚡ WSL2 Performance Tips

Windows users ke liye — Git 5-10x faster banayein

🐢
Problem: /mnt/d/ par Git bohat slow hota hai — Windows filesystem cross-access ki wajah se.
Solution: WSL Native Filesystem Use Karein 
# Project WSL native filesystem mein move karein
cp -r "/mnt/d/your/project" ~/your-project/
cd ~/your-project/

# Ab Git commands 5-10x fast chalengi
git add .
git commit -m "..."
git push
WSL Config Optimization 
# ~/.wslconfig (Windows user folder mein)

[wsl2]
memory=4GB
processors=4
swap=2GB
localhostForwarding=true
💡
Hamesha ~/ (WSL home) use karein, /mnt/d/ se bachein performance ke liye.

🚀 One-Liners

Copy-paste ready — complete workflows ek line mein

Naye project mein LFS setup
git lfs install && git lfs track "*.csv" "*.mat" "*.bin" && git add .gitattributes && git commit -m "chore: setup LFS"
Existing file ko LFS mein convert karein
git reset && git rm --cached large_file.bin && git add large_file.bin && git lfs ls-files | grep large_file
Remote change + push
git remote set-url origin https://github.com/user/repo.git && git push -u origin main
Full workflow — naya project (LFS ke saath)
git init && git lfs install && git lfs track "*.bin" && git add .gitattributes && git add . && git commit -m "init" && git remote add origin <URL> && git push -u origin main
Bash alias — quick access shortcut
# ~/.bashrc mein add karein
alias gitcheat='cat ~/git-lfs-cheatsheet.md'
alias gitlfs='git lfs'
source ~/.bashrc

🔐 GitHub Authentication

Personal Access Token (PAT) — setup aur usage

PAT Banana — Step by Step
  1. GitHub → Settings → Developer settings
  2. Personal access tokens → Tokens (classic)
  3. Generate new token → Note: "WSL-Git-LFS"
  4. Scopes select karein: repo, write:packages, delete:packages
  5. Generate → Token copy karein (sirf ek baar dikhega!)
PAT Use Karna 
# Jab bhi git push karein:
Username: your_github_username
Password: <paste_your_PAT_here>
# ❌ Normal password kaam nahi karega
# ✅ PAT hi use karein
🔒
Token ko secure jagah save karein — password manager ya encrypted note mein. GitHub par dubara nahi dikhega!

🎯 Pro Tips

01
.gitattributes hamesha commit karein — bina iske LFS rules doosre machines par apply nahi honge.
02
Code files (.py, .m, .js) ko LFS mein na daalein — unhein normal Git mein rehne dein.
03
Push se pehle git lfs ls-files zaroor check karein — large file track ho rahi hai ya nahi.
04
WSL users: /mnt/d/ drives se bach kar ~/ use karein performance ke liye — 5-10x speed up.
